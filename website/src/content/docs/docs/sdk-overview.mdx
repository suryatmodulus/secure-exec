`NodeRuntime` is the entry point: it boots a virtualized VM and runs guest
JavaScript end to end. This page is a map of what the SDK can do. Each section
shows a quick example and links to the feature guide for the full detail, and to
the [TypeScript SDK reference](/docs/sdks/typescript) for exact signatures and
option shapes.

## Your first run

Boot a runtime with `NodeRuntime.create()`, run some guest code, and dispose it.
Every `create()` option is optional.

```ts
import { NodeRuntime } from "secure-exec";

const rt = await NodeRuntime.create();
try {
  const { stdout } = await rt.exec("console.log('hi', 1 + 1)");
  console.log(stdout);
} finally {
  await rt.dispose();
}
```

*[See Full Example](https://github.com/rivet-dev/secure-exec/tree/main/examples/docs/sdk-overview)*

Read more in the [TypeScript SDK reference](/docs/sdks/typescript) for every `create()` option and method signature.

## Execute code

`exec()` runs guest code as an ES module and captures `stdout` / `stderr` /
`exitCode`; `run<T>()` also decodes a value the guest returns via
`globalThis.__return(value)`.

```ts
const { stdout } = await rt.exec(`console.log(2 + 2)`);
const { value } = await rt.run<{ sum: number }>(`
  globalThis.__return({ sum: 2 + 40 });
`);
console.log(value?.sum); // 42
```

Read more about [Executing Code](/docs/features/executing-code) and [Output Capture](/docs/features/output-capture).

## Long-running processes

`spawn()` starts a guest program (for example a dev server) and returns a live
handle you can stream, signal, and await, without waiting for it to exit.

```ts
const server = await rt.spawn(`
  import http from "node:http";
  http.createServer((_, res) => res.end("ok")).listen(3000);
`);
// ... later
server.kill();
await server.wait();
```

Read more about [Child Processes](/docs/features/child-processes).

## Resident runner

For tight loops of small snippets (an eval loop, a REPL), `createResidentRunner()`
keeps one guest process alive so repeated `exec` calls skip per-run startup and
run in about a millisecond.

```ts
const runner = await rt.createResidentRunner();
const a = await runner.exec("console.log(1 + 1)");
const b = await runner.exec("console.log(2 + 2)");
await runner.dispose();
```

Read more about [Resident Runner](/docs/features/resident-runner).

## Filesystem

The guest sees a fully virtualized in-memory filesystem; the host filesystem is
never exposed. Read and write the VFS from the host, seed files at boot, and
project host directories with mounts.

```ts
await rt.writeFile("/home/user/out.txt", "hello");
const bytes = await rt.readFile("/home/user/out.txt");
console.log(new TextDecoder().decode(bytes)); // "hello"
```

Read more about [Filesystem](/docs/features/filesystem) and [Filesystem Mounts](/docs/features/filesystem-mounts).

## NPM and modules

Guest `import` / `require` resolve real npm packages from a `node_modules` you
project into the VM, with naive Node resolution over that filesystem.

```ts
const rt = await NodeRuntime.create({
  nodeModules: "/abs/path/to/project/node_modules",
});
await rt.exec(`
  import isNumber from "is-number";
  console.log(isNumber(42)); // true
`);
```

Read more about [NPM & Module Loading](/docs/features/module-loading).

## Networking

Guest `fetch` and sockets run through the kernel socket table under the network
policy. Drive HTTP into a guest server with `rt.fetch(port, input)`, which works
even when guest egress is denied.

```ts
const res = await rt.fetch(3000, { path: "/health" });
console.log(res.status, res.body);
```

Read more about [Networking](/docs/features/networking).

## Host tools

Expose host-side functions the guest calls with the `callHostTool(name, input)`
global. The handler runs on the host, so you give untrusted code a curated
capability surface without granting the underlying access.

```ts
const rt = await NodeRuntime.create({
  tools: {
    add: {
      description: "Add two numbers",
      inputSchema: {
        type: "object",
        properties: { a: { type: "number" }, b: { type: "number" } },
        required: ["a", "b"],
      },
      handler: ({ a, b }: { a: number; b: number }) => ({ sum: a + b }),
    },
  },
});
await rt.exec(`
  const { sum } = await callHostTool("add", { a: 2, b: 3 });
  console.log(sum); // 5
`);
```

Read more about [Host Tools](/docs/features/host-tools).

## Permissions

The policy is secure by default: network access is denied while the virtualized
scopes stay enabled. Your policy merges over that default, so a partial policy
works.

```ts
const rt = await NodeRuntime.create({
  permissions: { network: "allow" },
});
```

Read more about [Permissions](/docs/features/permissions).

## Resource limits

Bound how long a guest can run with `timeout` and an `AbortSignal`; the VM also
enforces memory and CPU limits.

```ts
// A runaway loop is killed once the budget elapses.
const result = await rt.exec(`while (true) {}`, { timeout: 1000 });
console.log(result.exitCode); // non-zero: the guest was killed
```

Read more about [Resource Limits](/docs/features/resource-limits).

## Runtime and platform

Shape the host environment presented to guest JavaScript, starting with the env
and working directory the guest sees.

```ts
const rt = await NodeRuntime.create({
  env: { NODE_ENV: "production" },
  cwd: "/home/user",
});
```

Read more about [Runtime & Platform](/docs/features/runtime-platform).

## TypeScript

Compile and type-check TypeScript inside the sandbox with the in-VM compiler
tools.

```ts
import { createTypeScriptTools } from "@secure-exec/typescript";

const tools = createTypeScriptTools();
const { outputText } = await tools.compileSource({
  sourceText: "const x: number = 1 + 1;",
});
```

Read more about [TypeScript](/docs/features/typescript).

## Clean up

Always release the VM and its sidecar when done:

```ts
await rt.dispose();
```
